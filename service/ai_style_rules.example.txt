# Style rules for AI translation — one rule per line.
# Lines starting with # are ignored.
#
# Two ways to use this file (per-project takes priority over global):
#   • Per-project: copy it into an OmegaT project's root folder, named EXACTLY
#     "ai_style_rules.txt". The plugin loads it only under that exact name.
#   • Global default: copy it anywhere, fill in your rules, and point
#     STYLE_RULES_PATH in service/.env at that path.

# Example rules:
Use inclusive gender forms where applicable (e.g. "les membres" not gender-specific forms).
Use the median point and slashes to create gender-inclusive structures, e.g. " le/la directeur·trice·s" ou "les donateur·trice·s majeur·e·s"
In case of disagreement between translation memory entries and glossary entries, ensure compliance with the glossary.
